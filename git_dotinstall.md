# gitの利用目的
昔の変更履歴を管理
別機能を開発するために、異なるバージョンを同時並行で開発

# git における3つの状態
1. 作業ディレクトリ
1. ステージングエリア（インデックス）
1. リポジトリ

１→２　意味のあるまとまりで保存していく

# gitの利用設定(option)
`git config --global user.name "名前"`
`git config --global user.email "メールアドレス"`
設定確認
`git config -l`

# "git config" コマンドのヘルプをみたい時
`git config --help`
`git help config`

"help"部を別のコマンドに差し替えても
同様にヘルプ、オプションの参照が可能

# gitの基本的な流れ
初期化、コード作成（変更）、ステージング（add）、コミット
